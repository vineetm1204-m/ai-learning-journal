# Day 16 — CNN receptive field and spatial hierarchy

> 📅 July 07, 2026

---

## 🧠 Concept Explanation

A convolutional neural network builds a spatial hierarchy by stacking layers that each look at a local patch of the previous feature map. The receptive field of a neuron is the region in the input image that influences its activation. In the first layer, a 3×3 kernel sees only a 3×3 pixel patch. After a second 3×3 convolution (with stride 1), each unit aggregates information from a 5×5 region of the original image because the two kernels overlap. Pooling or strided convolutions enlarge the receptive field more aggressively: a 2×2 max‑pool doubles the field size. Deeper layers therefore capture larger, more abstract patterns — edges become textures, textures become object parts, and parts become whole objects. This hierarchy mirrors how the visual cortex processes information, allowing the network to recognize complex structures while preserving translation invariance. Understanding receptive field growth helps designers choose kernel sizes, depths, and strides to ensure the final layer covers the entire object of interest without excessive down‑sampling that would lose spatial detail.

---

## ✅ Key Points

- Receptive field = input region affecting a neuron's output.
- Stacking 3×3 convolutions grows the field linearly (3, 5, 7, …).
- Pooling or strided convolutions expand the field multiplicatively.
- Deeper layers capture larger, more abstract visual concepts.
- Design depth and stride so the final receptive field covers target objects.

## 🏗️ Diagram / Formula

```
Input (H×W) 
  │
  ├─ Conv3×3, s=1 → RF=3
  │
  ├─ Conv3×3, s=1 → RF=5
  │
  ├─ MaxPool2×2, s=2 → RF=10
  │
  └─ Conv3×3, s=1 → RF=14  (final layer)
```

## ⚠️ Common Mistakes

- Assuming a deeper network automatically sees the whole image without checking receptive field size.
- Using large kernels early on, which wastes parameters and reduces hierarchical abstraction.
- Over‑pooling so that the final feature map is too small to localize objects accurately.
