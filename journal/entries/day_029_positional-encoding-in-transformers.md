# Day 29 — Positional encoding in Transformers

> 📅 July 23, 2026

---

## 🧠 Concept Explanation

Transformers process all tokens in parallel, lacking the inherent sequential awareness of RNNs or CNNs. Positional encoding injects position information into token embeddings so the model can distinguish "dog bites man" from "man bites dog." The original Transformer uses fixed sinusoidal functions: for position pos and dimension i, PE(pos, 2i) = sin(pos / 10000^(2i/d_model)) and PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model)). These waves create unique signatures for each position while allowing the model to learn relative distances via linear relationships (e.g., PE(pos+k) can be expressed as a linear function of PE(pos)). The encodings are added to token embeddings before the first attention layer. Learned positional embeddings are an alternative, but sinusoidal versions generalize better to sequence lengths unseen during training. Without positional encoding, the self-attention mechanism would treat the input as a bag of words, destroying syntactic and semantic structure.

---

## ✅ Key Points

- Transformers have no built-in notion of token order; positional encoding supplies it.
- Sinusoidal encodings use sine/cosine waves of geometrically increasing wavelengths.
- Encodings are added to token embeddings (not concatenated) before the first layer.
- The formulation enables extrapolation to longer sequences via linear relations.
- Learned positional embeddings are a trainable alternative but may overfit to training lengths.

```
PE(pos, 2i)     = sin(pos / 10000^(2i / d_model))

Input Embedding  +  Positional Encoding  =  Transformer Input
```

## 💡 Real-World Analogy

Like page numbers in a book: the words (tokens) carry meaning, but the page numbers (positional encodings) tell you the order so the story makes sense.

---

## ⚠️ Common Mistakes

- Forgetting to add positional encodings, causing the model to ignore sequence order.
- Using learned positional embeddings without handling variable-length sequences at inference.
- Concatenating instead of adding positional encodings, unnecessarily increasing dimensionality.
