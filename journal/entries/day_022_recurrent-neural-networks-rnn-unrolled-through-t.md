# Day 22 — Recurrent Neural Networks (RNN) unrolled through time

> 📅 July 14, 2026

## 🧠 Concept Explanation

Recurrent Neural Networks (RNNs) process sequences by maintaining a hidden state that acts as memory. To understand how they learn over time, we 'unroll' the network: we replicate the same cell for each time step, creating a deep feedforward network where each layer shares identical weights. At step t, the cell takes input x_t and the previous hidden state h_{t-1}, computes a new state h_t, and produces output y_t. This unrolled view reveals that the forward pass is just a chain of function compositions, and the backward pass becomes Backpropagation Through Time (BPTT), where gradients flow backward through the unrolled steps. Because the same parameters (W_hh, W_xh, W_hy) are used at every step, the total gradient for a weight is the sum of gradients from each time step. This weight sharing drastically reduces parameters compared to a standard deep network but creates long dependency paths. The unrolled graph makes it clear why vanilla RNNs struggle with long sequences: gradients must traverse many matrix multiplications, leading to vanishing or exploding gradients. Modern architectures like LSTMs and GRUs introduce gating mechanisms to create shortcut paths for gradient flow, effectively learning what to remember and what to forget across the unrolled depth.

---

## ✅ Key Points

- Unrolling transforms the cyclic RNN graph into an acyclic deep network with shared weights across time steps.
- Forward pass computes h_t = f(W_hh * h_{t-1} + W_xh * x_t + b_h) and y_t = g(W_hy * h_t + b_y).
- Backpropagation Through Time (BPTT) sums gradients from all time steps for each shared parameter.
- Vanishing/exploding gradients arise from repeated multiplication of the same weight matrix (W_hh) over many steps.
- Gradient clipping and gated cells (LSTM/GRU) are standard solutions to stabilize training in unrolled RNNs.

```
                  |           |           |           |           |
                  v           v           v           v           v
                 y_{t-1}     y_t         y_{t+1}

Shared Weights: W_xh, W_hh, W_hy (same matrices in every box)
```

## 💡 Real-World Analogy

Unrolling an RNN is like reading a book one page at a time: you carry your understanding (hidden state) from the previous page to the next, and the same reading skills (shared weights) apply to every page.

---

## ⚠️ Common Mistakes

- Forgetting to detach the hidden state between batches, causing gradients to flow across unrelated sequences.
- Using a too-large learning rate without gradient clipping, leading to exploding gradients in deep unrolled graphs.
- Assuming the unrolled network has independent parameters per layer, missing the weight-sharing constraint that defines RNNs.
