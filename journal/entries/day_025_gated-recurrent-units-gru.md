# Day 25 — Gated Recurrent Units (GRU)

## 🧠 Concept Explanation

Gated Recurrent Units (GRUs) are a streamlined variant of Long Short-Term Memory networks designed to capture temporal dependencies with fewer parameters. Like an LSTM, a GRU maintains a hidden state that flows through time, but it merges the forget and input gates into a single update gate and adds a reset gate that controls how much past information to discard before computing the new candidate activation. At each time step t, the update gate z_t decides how much of the previous hidden state h_{t-1} to keep versus how much of the new candidate 	ilde{h}_t to incorporate. The reset gate r_t determines whether the previous hidden state should influence the candidate, allowing the model to effectively “reset” its memory when the context changes. Because GRUs have only two gates and no separate cell state, they are computationally cheaper and often train faster while achieving comparable performance on many sequence tasks such as language modeling, speech recognition, and time‑series forecasting. The update gate uses a sigmoid to produce values between 0 and 1, enabling smooth interpolation between old and new information. The reset gate similarly uses a sigmoid; when near zero the candidate ignores the previous hidden state, starting a fresh representation. This gating mitigates vanishing gradients by providing direct gradient paths across many steps. Empirically, GRUs often match LSTM accuracy with ~30% fewer parameters, suiting resource‑constrained deployments.

## ✅ Key Points

- GRU merges forget and input gates into a single update gate.
- Reset gate controls how much previous hidden state contributes to the candidate activation.
- Only two gates (update and reset) and no separate cell state.
- Fewer parameters and faster training than LSTM.
- Effectively mitigates vanishing gradients with comparable performance.

## 🏗️ Diagram / Formula

```
x_t ──►│
       ├─► r_t = σ(W_r·[h_{t-1}, x_t])
       ├─► z_t = σ(W_z·[h_{t-1}, x_t])
       └─► \tilde{h}_t = tanh(W·[r_t ⊙ h_{t-1}, x_t])
h_t = (1 - z_t) ⊙ h_{t-1} + z_t ⊙ \tilde{h}_t
```

## 💡 Real-World Analogy

A GRU works like a notebook where the update gate decides how much of the old notes to keep versus new entries, and the reset gate lets you tear out irrelevant pages before writing the next entry.

---

## ⚠️ Common Mistakes

- Confusing GRU gates with LSTM's three separate gates.
- Forgetting the element‑wise multiplication (⊙) when applying the reset gate.
- Assuming GRU always outperforms LSTM and skipping task‑specific hyper‑parameter tuning.
