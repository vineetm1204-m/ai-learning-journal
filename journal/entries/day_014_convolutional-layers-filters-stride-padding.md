# Day 14 — Convolutional layers: filters, stride, padding

## 🧠 Concept Explanation

Convolutional layers are the workhorse of modern computer vision. They apply a set of learnable filters (also called kernels) across an input volume—typically an image with height, width, and channel dimensions. Each filter slides over the spatial dimensions, computing a dot product between its weights and the local patch of the input, producing a single scalar for that position. The collection of these scalars forms a feature map. Stride controls how many pixels the filter jumps at each step; a stride of 1 moves the filter one pixel at a time, while larger strides down‑sample the output. Padding adds extra border pixels (usually zeros) around the input so that the output can retain the same spatial size as the input or to control the receptive field. By stacking many convolutional layers, the network builds hierarchical representations: early layers detect edges and textures, deeper layers compose them into shapes and objects. Because the same filter is reused across all spatial locations, convolution dramatically reduces the number of parameters compared to a fully connected layer, enabling translation invariance and efficient learning on high‑resolution data. The number of filters determines the depth of the output volume, and each filter learns a distinct pattern. During back‑propagation, gradients flow through the same shared weights, reinforcing useful features across the entire image. This weight sharing is the key to the parameter efficiency of CNNs.

---

## ✅ Key Points

- Filters (kernels) slide over input to produce feature maps.
- Stride controls step size and spatial down‑sampling.
- Padding preserves spatial dimensions or adjusts receptive field.
- Weight sharing reduces parameters and gives translation invariance.
- Stacking layers builds hierarchical representations.

## 💡 Real-World Analogy

A convolutional filter is like a flashlight scanning a wall: the beam (filter) reveals local patterns (edges, textures) as it moves across the surface, and the stride is how far you step each time.

---

## ⚠️ Common Mistakes

- Forgetting to account for padding when computing output size.
- Using too large a stride, losing spatial information.
- Treating each filter as independent per location (ignoring weight sharing).
