# Day 6 — Weight initialization strategies: Xavier, He, random

> 📅 June 20, 2026

## 🧠 Concept Explanation

Weight initialization sets the starting values of a neural network’s parameters before training begins. If weights are too large, activations explode and gradients vanish; if too small, signals die out and learning stalls. Xavier (Glorot) initialization draws weights from a zero‑mean Gaussian with variance 2/(n_in + n_out) (or a uniform range ±√6/(n_in+n_out)), aiming to keep the variance of activations and back‑propagated gradients roughly constant across layers for linear or tanh activations. He initialization adapts this idea for ReLU‑type nonlinearities by using variance 2/n_in (or uniform ±√6/n_in), because ReLU zeroes out half the inputs, effectively halving the forward variance. Random uniform or normal initialization with a fixed small scale (e.g., 0.01) works for very shallow nets but fails in deep architectures. Proper initialization reduces the number of epochs needed for convergence, mitigates vanishing/exploding gradients, and makes training more stable across architectures and batch sizes. In practice, modern libraries default to He for layers followed by ReLU and Xavier for sigmoid or tanh. When using batch normalization, the exact scale matters less because normalization rescales activations, but good initialization still speeds up the first few epochs. For recurrent networks, orthogonal or identity initialization preserves gradient flow over many time steps. Choosing the right scheme is a low‑cost hyper‑parameter that can dramatically affect final performance.

## ✅ Key Points

- Xavier/Glorot keeps variance constant for linear/tanh using 2/(n_in+n_out).
- He initialization uses 2/n_in for ReLU because half activations zero.
- Random small fixed scale works only for shallow nets.
- Proper init reduces vanishing/exploding gradients and speeds convergence.
- Batch norm lessens sensitivity but good init still helps early training.

## 🏗️ Diagram / Formula

```
Xavier (Gaussian):   W ~ N(0, 2/(n_in + n_out))
He (Gaussian):       W ~ N(0, 2/n_in)
Uniform variants:    W ~ U(-√6/(n_in+n_out), √6/(n_in+n_out))
                     W ~ U(-√6/n_in, √6/n_in)
```

## 💡 Real-World Analogy

Like tuning a guitar before a concert: if strings are too tight they snap (exploding gradients), too loose they produce no sound (vanishing gradients); proper initialization finds the sweet tension for each layer.

---

## ⚠️ Common Mistakes

- Using the same tiny constant (e.g., 0.01) for all layers in deep networks.
- Applying Xavier to ReLU layers instead of He.
- Ignoring initialization for recurrent or transformer layers, causing unstable training.
