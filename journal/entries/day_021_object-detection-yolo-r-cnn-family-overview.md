# Day 21 — Object detection: YOLO, R-CNN family overview

> 📅 July 12, 2026

## 🧠 Concept Explanation

Object detection localizes and classifies multiple objects in an image. The R‑CNN family (R‑CNN, Fast R‑CNN, Faster R‑CNN) pioneered a two‑stage pipeline: first a region proposal network (or selective search) suggests candidate boxes, then a CNN classifies each proposal and refines its coordinates. This yields high accuracy but incurs latency because each proposal is processed separately. YOLO (You Only Look Once) reframes detection as a single regression problem. The image is divided into an S×S grid; each grid cell predicts B bounding boxes, confidence scores, and class probabilities in one forward pass. Subsequent versions (YOLOv2‑v8) add anchor boxes, multi‑scale predictions, and improved backbones, dramatically increasing speed while narrowing the accuracy gap. Single‑stage detectors like SSD and RetinaNet follow a similar philosophy, using feature‑pyramid networks to handle scale. The trade‑off is clear: two‑stage methods excel at precise localization and small objects, whereas single‑stage models prioritize real‑time inference. Modern practice often blends ideas — e.g., using a lightweight backbone with a region‑proposal head for a balanced speed/accuracy profile.

---

## ✅ Key Points

- R‑CNN family uses region proposals + per‑proposal CNN classification (two‑stage).
- Faster R‑CNN introduces a learnable Region Proposal Network (RPN) for end‑to‑end training.
- YOLO treats detection as a single regression: grid cells predict boxes, confidence, classes in one pass.
- Anchor boxes and multi‑scale feature maps let single‑stage detectors handle varied object sizes.
- Two‑stage detectors generally achieve higher mAP; single‑stage detectors achieve higher FPS.

## 💡 Real-World Analogy

R‑CNN is like a detective who first marks every suspicious spot on a map, then investigates each spot individually; YOLO is a security guard who scans the whole scene at once and instantly flags threats.

---

## ⚠️ Common Mistakes

- Treating IoU thresholds as hyper‑parameters for training loss instead of only for evaluation.
- Using a single‑scale anchor set for datasets with large scale variance, causing missed small objects.
- Ignoring the need for non‑maximum suppression (NMS) post‑processing, leading to duplicate detections.
