## 🧠 Concept Explanation

Gradient descent is the workhorse optimizer that iteratively nudges model parameters toward lower loss. In batch gradient descent the gradient is computed on the entire training set, giving a precise but often expensive direction. Stochastic gradient descent (SGD) uses a single randomly chosen example per step, producing a noisy but cheap update that can escape shallow local minima. Mini‑batch gradient descent sits in between: it averages gradients over a small subset (typically 32‑256 samples), balancing stability and speed. All three follow the same core rule θ ← θ − η∇L, where η is the learning rate and ∇L the gradient estimate. The choice influences convergence speed, memory usage, and generalization. Batch descent converges smoothly but may stall on large datasets; SGD converges quickly early on but oscillates near the optimum; mini‑batch offers a practical sweet spot and works well with vectorized hardware. Adaptive methods (Adam, RMSprop) build on these schemes by scaling the learning rate per parameter. Understanding the trade‑offs helps you pick the right variant for your data size, hardware, and desired training dynamics. In practice, mini‑batch sizes are tuned to fit GPU memory while keeping gradient variance low, and learning‑rate schedules further shape the trajectory. Monitoring validation loss helps detect divergence early.

---

## ✅ Key Points

- Batch GD uses the full dataset for each gradient step, giving low variance but high compute cost.
- SGD updates on a single sample, yielding high variance updates that can escape local minima.
- Mini‑batch GD averages over a small subset, offering a good trade‑off between speed and stability.
- All variants share the update rule θ ← θ − η∇L; only the gradient estimate changes.
- Choosing batch size affects hardware utilization, convergence speed, and generalization.

## 🏗️ Diagram / Formula

```
Batch GD:      ∇L = (1/N) Σ_i ∇ℓ(θ; x_i, y_i)
Mini‑batch GD: ∇L = (1/B) Σ_{i∈batch} ∇ℓ(θ; x_i, y_i)
SGD:           ∇L = ∇ℓ(θ; x_k, y_k)   (k random)
Update:        θ ← θ − η ∇L
```

## 💡 Real-World Analogy

Batch GD is like steering a ship with a precise map of the whole ocean; SGD is like navigating by looking at one wave at a time; mini‑batch GD checks a small cluster of waves to decide the next turn.

---

## ⚠️ Common Mistakes

- Using a too‑large learning rate with SGD, causing divergence.
- Setting mini‑batch size larger than GPU memory, leading to out‑of‑memory errors.
- Assuming batch GD always converges faster; on huge data it can be impractically slow.
