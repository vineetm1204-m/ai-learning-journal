# Day 18 — ResNets and skip connections

> 📅 July 09, 2026

## 🧠 Concept Explanation

Residual Networks (ResNets) solve the degradation problem that appears when very deep plain networks are trained: accuracy saturates and then degrades even though the deeper model should be able to represent the shallower one. The core idea is to let layers learn a residual function F(x) = H(x) - x instead of the full mapping H(x). By adding the original input x to the output of a few stacked layers via a shortcut (skip) connection, the network can easily learn an identity mapping (F(x)=0) if the extra layers are unnecessary. This makes optimization much easier because gradients can flow directly through the shortcut, mitigating vanishing gradients. In practice, a residual block consists of two or three convolutional layers with batch normalization and ReLU, followed by an element‑wise addition of the block's input. If dimensions change, a 1×1 convolution projects the shortcut to match. Stacking many such blocks yields networks with 50, 101, or even 152 layers that train reliably and achieve state‑of‑the‑art performance on image classification, detection, and segmentation tasks.

---

## ✅ Key Points

- Skip connections add the input directly to the output of a layer stack, enabling identity mappings.
- They alleviate vanishing gradients by providing a gradient highway through the network.
- Residual blocks typically use 2‑3 convolutions with batch norm and ReLU before the addition.
- When dimensions differ, a 1×1 projection (or stride) aligns the shortcut.
- Deep ResNets (50‑152 layers) train faster and achieve higher accuracy than equally deep plain nets.

## 🏗️ Diagram / Formula

```
   x ──► [Conv+BN+ReLU] ──► [Conv+BN] ──► +
                     │                │          │
                     └───── shortcut ──┘          ▼
                                            [ReLU] ──► out
```

## 💡 Real-World Analogy

Think of a relay race where each runner can either pass the baton normally or hand it directly to the finish line; the direct hand‑off (skip connection) ensures the baton never gets lost even if some runners stumble.

---

## ⚠️ Common Mistakes

- Forgetting to match tensor shapes on the shortcut, causing runtime errors.
- Placing the ReLU before the addition instead of after, which blocks gradient flow.
- Using too many residual blocks without increasing width, leading to over‑parameterization and slow training.
