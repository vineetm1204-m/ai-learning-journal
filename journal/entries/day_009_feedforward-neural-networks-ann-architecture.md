## 🧠 Concept Explanation

A feedforward neural network (FNN), often called a multi-layer perceptron (MLP), is the foundational architecture of deep learning. Information flows strictly in one direction — from input layer, through one or more hidden layers, to the output layer — with no cycles or loops. Each layer consists of neurons (units) that compute a weighted sum of their inputs, add a bias term, and apply a non-linear activation function (e.g., ReLU, sigmoid, tanh). This non-linearity is crucial; without it, the entire network would collapse to a single linear transformation, unable to model complex patterns. The input layer receives raw features (e.g., pixel values), hidden layers learn hierarchical representations (edges → shapes → objects), and the output layer produces predictions (class probabilities, regression values). Training uses backpropagation with gradient descent to minimize a loss function by adjusting weights and biases. Universal approximation theorem guarantees that a sufficiently wide FNN with at least one hidden layer can approximate any continuous function, though depth often achieves this more efficiently. Key design choices include number of layers, neurons per layer, activation functions, weight initialization, and regularization (dropout, weight decay) to prevent overfitting.

---

## ✅ Key Points

- Unidirectional flow: input → hidden layers → output, no feedback loops
- Non-linear activations (ReLU, sigmoid) enable learning complex functions
- Universal approximation: one hidden layer can approximate any continuous function
- Backpropagation computes gradients efficiently via chain rule for weight updates
- Depth vs. width trade-off: deeper networks often learn hierarchical features more efficiently

## 💡 Real-World Analogy

Like a factory assembly line: raw materials (input) pass through specialized workstations (hidden layers), each refining the product, until the finished item (output) emerges — no station sends items backward.

---

## ⚠️ Common Mistakes

- Using linear activations in hidden layers, reducing the network to a single linear model
- Choosing excessively large layers without regularization, causing severe overfitting
- Ignoring input normalization, leading to unstable gradients and slow convergence
