# Day 7 — Regularization: L1, L2, Dropout, BatchNorm

## 🧠 Concept Explanation

Regularization is the art of preventing a model from memorizing training data so it generalizes to unseen examples. L1 and L2 add penalty terms to the loss function: L1 (Lasso) pushes weights toward exact zero, yielding sparse models useful for feature selection; L2 (Ridge) shrinks weights proportionally, keeping all features but reducing their influence. Dropout randomly deactivates a fraction of neurons during each forward pass, forcing the network to learn redundant, robust representations instead of relying on specific pathways. Batch Normalization standardizes layer inputs to zero mean and unit variance per mini-batch, stabilizing gradients and allowing higher learning rates; it also acts as a mild regularizer by injecting noise through batch statistics. Together, these techniques address overfitting from different angles: weight penalties constrain model complexity, dropout breaks co-adaptation, and BatchNorm smooths the optimization landscape. In practice, L2 and Dropout are staples in fully connected layers, while BatchNorm dominates convolutional architectures. Choosing the right mix depends on dataset size, model depth, and whether interpretability (L1) or raw performance (L2+Dropout+BN) is the priority.

---

## ✅ Key Points

- L1 regularization induces sparsity by driving weights to zero, enabling feature selection.
- L2 regularization shrinks weights uniformly, preventing any single feature from dominating.
- Dropout randomly zeros activations during training, forcing robust distributed representations.
- BatchNorm normalizes layer inputs per mini-batch, accelerating convergence and adding slight regularization.
- Combining methods (e.g., L2 + Dropout + BatchNorm) often yields better generalization than any single technique.

## 🏗️ Diagram / Formula

```
L1 Loss:  L = L_task + λ Σ |w_i|
L2 Loss:  L = L_task + λ Σ w_i²

Dropout:  y = x * Bernoulli(1-p) / (1-p)  (inverted dropout scaling)

BatchNorm:  x̂ = (x - μ_B) / √(σ_B² + ε)
            y = γ x̂ + β  (learnable scale γ, shift β)
```

## 💡 Real-World Analogy

Regularization is like a teacher who prevents students from memorizing the textbook by occasionally erasing words (Dropout), penalizing overly complex answers (L1/L2), and standardizing grading curves across classrooms (BatchNorm).

---

## ⚠️ Common Mistakes

- Applying dropout at inference time instead of scaling weights, causing under-confident predictions.
- Using L1 regularization expecting feature selection in deep networks where features are distributed across layers.
- Placing BatchNorm after the activation function, which breaks its variance-stabilizing assumption.
