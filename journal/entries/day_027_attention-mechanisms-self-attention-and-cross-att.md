# Day 27 — Attention mechanisms: self-attention and cross-attention

> 📅 July 19, 2026

---

## 🧠 Concept Explanation

Attention mechanisms allow neural networks to dynamically weight the importance of different parts of an input sequence when producing an output. Self-attention, the core of Transformers, computes relationships between all positions within a single sequence. For each token, it generates query, key, and value vectors. The attention score between a query and all keys is calculated via scaled dot-product, then softmaxed to produce a probability distribution. This distribution weights the value vectors, yielding a context-aware representation for each token. Cross-attention extends this idea to two different sequences: queries come from one sequence (e.g., decoder), while keys and values come from another (e.g., encoder). This enables the model to align and fuse information across modalities or languages, as in machine translation where the decoder attends to encoder states. Multi-head attention runs several attention heads in parallel, each learning different relational patterns, then concatenates and projects them. The result is a powerful, parallelizable alternative to recurrence that captures long-range dependencies without sequential bottlenecks. By making the weighting explicit and differentiable, attention provides interpretability — we can visualize what the model focuses on — and has become the backbone of modern large language models, vision transformers, and multimodal systems.

---

## ✅ Key Points

- Self-attention computes token-to-token relationships within a single sequence using Q, K, V projections.
- Cross-attention lets one sequence attend to another, enabling encoder-decoder fusion.
- Scaled dot-product attention uses softmax(QK^T/√d_k)V for stable gradients.
- Multi-head attention captures diverse dependency patterns in parallel.
- Attention replaces recurrence, enabling full parallelization and long-range context.

## 🏗️ Diagram / Formula

```
Self-Attention:
  X -> Linear -> Q, K, V
  Scores = softmax(Q @ K^T / sqrt(d_k))
  Output = Scores @ V

Cross-Attention:
  Decoder_X -> Linear -> Q
  Encoder_X -> Linear -> K, V
  Scores = softmax(Q @ K^T / sqrt(d_k))
  Output = Scores @ V
```

## 💡 Real-World Analogy

Self-attention is like a group discussion where everyone listens to everyone else to update their understanding; cross-attention is like a student (decoder) referring to a textbook (encoder) while writing an essay.

---

## ⚠️ Common Mistakes

- Confusing self-attention with cross-attention: using the same sequence for Q, K, V when two distinct sequences are needed.
- Forgetting to scale dot-products by √d_k, causing large gradients and training instability.
- Applying attention without masking in autoregressive decoding, allowing future tokens to leak information.
